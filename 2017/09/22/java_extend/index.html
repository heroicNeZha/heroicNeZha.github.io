Java 面向对象编程(中)

继承

  • 提高了代码的复用性
  • 让类与类之间产生了关系,为第三个特征多态提供了前提
  • 所有枚举类型都是Enum的子类,它提供了toString,valueOf和CompareTo方法
  • Java支持单继承, 不直接支持多继承. 多继承的的弊端是方法调用具有不确定性,因为方法的主体不同,Java对其进行了改良;
  • 当事物存在(is-a)的关系时, 可以通过继承实现这个关;
  • 子父类出现的相同名字的成员变量, 用super关键字来区分;

  • 子类每一个构造函数第一行都有隐式的super();
    如果父类没有空参数构造函数, 则需要显式调用super的方式调用父类构造函数;
    并且只能定义在第一行, 因为父类的初始化要先完成;
    super()和this()不能同时出现,因为他们只能定义在第一行;

  • 将子类对象赋给父类变量是合法的

现在考虑一下，当在父类变量上调用一个方法时会发生什么。

即使empl的类型是Employee，还是会调用Manager.getSalary()方法。
当调用方法时，虚拟机会查看对象的实际类型，并且定位方法的版本。这个过程被称作动态方法查找

  • 不同于this引用，super不是对象的引用，而是绕过动态查找方法并调用特定方法的指令

万物基类–Object

  • 所有类都隐式或显示的继承了Object类, 而Object类定义了7个适用于所有java对象的方法
方法 描述
String toString() 产生一个表示该对象的字符串,默认该字符串是类名称和哈希码
boolean equals(Object other) 如果other被认为与这个对象相等,返回true;如果other为null或者不相等,返回false. 如果两者指向同一对象,返回true. 如果想在other为null的时候也是安全的,推荐使用Object.equals(Object obj, Object other)
Class<？> getClass() 产生该对象所属的类的Class对象
protected Object clone() 拷贝该对象，默认为浅拷贝
protected void finalize() 垃圾回收机回收时调用，不要覆盖这个方法
wait、notify、notifyAll 并发编程
