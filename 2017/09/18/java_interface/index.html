Java 接口

接口是服务提供方和想让他们的对象对服务是可用的那些类，之间约定合同的一种机制。

  • 实现接口

    实体类必须实现接口的所有方法，但抽象类可以只实现部分方法

  • 虽然可以声明接口类型的变量，但不能存在类型为接口的对象，所有的对象都是类的实例
  • 想从父类转换为子类可以使用强制类型转换(cast)，但弄错了就会发生编译时错误或强制类型转换异常
  • 为了避免异常可以使用instanceof操作符来测试对象是否是期望的类型
  • 接口可以被继承，实现子接口的类必须实现子父接口中的所有方法
  • 一个类可以实现很多个接口，但不能继承多个类
  • 定义在接口中的任何变量都自动为public static final

    接口中的方法实现

  • 接口中可以有静态方法与默认方法，过去或许将接口作为抽象规范，但现在思想变了
  • 静态方法常常用作工厂方法

  • 默认方法则提供一个默认实现

  • 默认方法的一个重要用途是接口演化，比如为接口Collection添加一个新方法

那Collection的实现类Bag将无法通过编译，因为它们没有实现新方法。但如果提供了默认实现，就可以向后兼容了。这时编译后在Bag的实例上调用新方法，就会自动寻找Collection的默认实现了

  • 如果有两个接口,它们具有相同的方法名并且至少有一个提供了默认实现,那么实现这两个接口的类就会发生冲突，而解决办法可以使用委托其中一个冲突方法或者提供自己的方法并实现计划(此条存疑,经试验为自动实现第一个接口的方法)
