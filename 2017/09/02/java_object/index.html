Java 面向对象编程(上)

面向对象编程

  • 面向对象三个特征:封装,继承和多态
  • (Accessor)访问器和(Mutator)更改器方法,其中更改器方法更好

final关键字

  • 当你将一个方法声明为final时，子类不可以覆盖它。声明为final的类不能被继承。
  • 有final声明的变量必须在所有的构造函数末尾初始化。之后该变量不会被再次修改
  • final声明引用时，只是声明该引用永不改变，但是修改对象自身是完全合法的 来，比如数组列表添加元素

static关键字

  • 如果你在类中将变量声明为static，那么该变量是属于这个类的，而不是对象

  • 可修改的静态常量很少，但是静态常量(static final)很普遍，比如System.out，它在System类中这样声明

  • 静态初始块，用于额外的静态初始化工作

  • 静态方法，同样使用static修饰，用于给不属于你的类提供额外功能等
  • 不能在静态方法中访问实例变量，因为静态方法没有运行在对象上
  • 静态方法最常见的使用就是工厂方法，返回类的新实例的静态方法，工厂方法也能返回共享对象

封装

  • 封装就是隐藏实现细节. 对外提供公共的访问方式(接口).
  • 封装的目的是为了解耦, 降低耦合可以加快系统的开发速度, 减轻维护的负担. 还提高了软件的可重用行,
    也便于进一步优化.
  • 访问控制机制. 实体的可访问性由该实体声明所在的位置,以及该实体声明中所出现的访问修饰符
    (private,protected,public)共同决定. 第一规则是尽可能地使每个类或者成员不被外界访问.
创建对象时的流程如下:
  1. 加载指定的字节码文件进内存
  2. 通过new在堆内存中开辟空间并分配首地址值
  3. 对象中的属性进行默认初始化
  4. 调用对应的构造函数. 构造函数压栈
  5. 隐式执行super()访问父类构造函数
  6. 对象中的属性进行初始化
  7. 调用类中的构造代码块
  8. 执行构造函数中自定义的初始化代码
  9. 初始化完毕, 将地址赋给指定的引用
