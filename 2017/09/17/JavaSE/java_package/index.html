Java 包

包

  • 使用包的主要原因是保证类名的唯一性，比如两个程序都提供了一个Element方法，只要开发者将他们开发的类放到不同的包中，就没有冲突
  • 包的声明使用以点号分隔的标识符，例如Java.util.regex
  • 为了保证包名的唯一性，可以使用倒着写的因特网域名比如com.lhaojing.javaProgram，但Java的标准类库的包名是以Java或者javax开始的
  • 在Java中，包不能嵌套，Java.util和java.util.regex是各自独立类的集合
  • 从文件系统中读取类文件时，路径名称必须匹配包名称

类路径

  • 你可以将class文件放到一个或多个被称为JAR文件的归档文件中，而不是将class文件存储在文件系统，可以使用jar工具制作归档文件，它的命令行参数与Unix上的tar程序类似

    包类库常常这样做

  • 也可以使用JAR将程序打包，而不仅仅是类库，运行时输入

  • 当你在项目中使用类库的JAR时需要指定class path来告诉编译器和虚拟机这些文件的位置

  • 注意：Windows下分隔符为分号，Unix下为冒号
  • 设置类路径首选使用-classpath，如果设置全局变量未来可能找不到自己的类

包作用域

  • 不指定public和private的类可以被包内所有方法访问
  • 一个源文件可以包含很多个类，但是最多只有一个可以被声明为public，如果一个源文件有个public类，那么文件名和类名必须匹配
  • 变量的默认作用域是包级，比如java.awt包中window类里面，warningString变量就因为忘记了private修饰符从而可以被awt包中任何类访问
  • 可以通过将自己的类放到封闭的jar文件中，并添加manifest来保护自己的包

嵌套类

  • 静态嵌套类的作用有限制可见性或避免因为导入一些使用了通用名称的包而产生混乱
  • 内部类相当于静态嵌套类去掉了static，这时它可以访问他的外部类的实例变量，这种情况下他就是外部类对象的实例变量，每个内部类对象都有自己外部类对象outer的隐藏引用
  • 当嵌套类的实例不需要知道它属于外部类的哪个实例时，使用静态嵌套类。当这些信息重要时，使用内部类
  • outer实际上应该是OuterClass.this，但这不是必须的
  • 内部类不应该有静态成员，不然static的含义界限很模糊
