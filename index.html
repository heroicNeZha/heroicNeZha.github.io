  • 枚举的定义

  • 枚举类型的静态方法valueOf很常用，它针对枚举类型而合成。例如

将Size.SMALL赋给notMySize。如果给定名称的实例不存在，valueOf会出IllegalArgumentException异常

  • 每个枚举类型都有一个静态方法values，该方法返回按照其声明次序排序的包含所有枚举实例的数组

这个方法也能用于foreach遍历

  • 可以给枚举类型添加构造函数、方法和域。每个枚举类型实例保证只被构造一次

  • 从技术上说，每个枚举常量都是枚举类的一个匿名子类，匿名子类体中的东西当然可以添加实现

  • 枚举可以有静态成员，但需要使用静态初始化块来进行初始化工作，因为枚举常量在静态成员之前构建

  • 枚举类型可以被嵌套在一个类中，这样嵌套的枚举类属于隐式的静态嵌套类，所以一样不能引用外部类的实例变量

  • 枚举常量可以被switch

Java 包

包

  • 使用包的主要原因是保证类名的唯一性，比如两个程序都提供了一个Element方法，只要开发者将他们开发的类放到不同的包中，就没有冲突
  • 包的声明使用以点号分隔的标识符，例如Java.util.regex
  • 为了保证包名的唯一性，可以使用倒着写的因特网域名比如com.lhaojing.javaProgram，但Java的标准类库的包名是以Java或者javax开始的
  • 在Java中，包不能嵌套，Java.util和java.util.regex是各自独立类的集合
  • 从文件系统中读取类文件时，路径名称必须匹配包名称

类路径

  • 你可以将class文件放到一个或多个被称为JAR文件的归档文件中，而不是将class文件存储在文件系统，可以使用jar工具制作归档文件，它的命令行参数与Unix上的tar程序类似

    包类库常常这样做

  • 也可以使用JAR将程序打包，而不仅仅是类库，运行时输入

  • 当你在项目中使用类库的JAR时需要指定class path来告诉编译器和虚拟机这些文件的位置

  • 注意：Windows下分隔符为分号，Unix下为冒号
  • 设置类路径首选使用-classpath，如果设置全局变量未来可能找不到自己的类

包作用域

  • 不指定public和private的类可以被包内所有方法访问
  • 一个源文件可以包含很多个类，但是最多只有一个可以被声明为public，如果一个源文件有个public类，那么文件名和类名必须匹配
  • 变量的默认作用域是包级，比如java.awt包中window类里面，warningString变量就因为忘记了private修饰符从而可以被awt包中任何类访问
  • 可以通过将自己的类放到封闭的jar文件中，并添加manifest来保护自己的包

静态导入

  • 可以使用import来导入静态的方法和变量比如

这样就可以直接使用Math类的静态方法和变量了

嵌套类

  • 静态嵌套类的作用有限制可见性或避免因为导入一些使用了通用名称的包而产生混乱
  • 内部类相当于静态嵌套类去掉了static，这时它可以访问他的外部类的实例变量，这种情况下他就是外部类对象的实例变量，每个内部类对象都有自己外部类对象outer的隐藏引用
  • 当嵌套类的实例不需要知道它属于外部类的哪个实例时，使用静态嵌套类。当这些信息重要时，使用内部类
  • outer实际上应该是OuterClass.this，但这不是必须的
  • 内部类不应该有静态成员，不然static的含义界限很模糊

Java 面向对象编程(上)

面向对象编程

  • 面向对象三个特征:封装,继承和多态
  • (Accessor)访问器和(Mutator)更改器方法,其中更改器方法更好

final关键字

  • 当你将一个方法声明为final时，子类不可以覆盖它。声明为final的类不能被继承。
  • 有final声明的变量必须在所有的构造函数末尾初始化。之后该变量不会被再次修改
  • final声明引用时，只是声明该引用永不改变，但是修改对象自身是完全合法的 来，比如数组列表添加元素

static关键字

  • 如果你在类中将变量声明为static，那么该变量是属于这个类的，而不是对象

  • 可修改的静态常量很少，但是静态常量(static final)很普遍，比如System.out，它在System类中这样声明

  • 静态初始块，用于额外的静态初始化工作

  • 静态方法，同样使用static修饰，用于给不属于你的类提供额外功能等
  • 不能在静态方法中访问实例变量，因为静态方法没有运行在对象上
  • 静态方法最常见的使用就是工厂方法，返回类的新实例的静态方法，工厂方法也能返回共享对象

封装

  • 封装就是隐藏实现细节. 对外提供公共的访问方式(接口).
  • 封装的目的是为了解耦, 降低耦合可以加快系统的开发速度, 减轻维护的负担. 还提高了软件的可重用行,
    也便于进一步优化.
  • 访问控制机制. 实体的可访问性由该实体声明所在的位置,以及该实体声明中所出现的访问修饰符
    (private,protected,public)共同决定. 第一规则是尽可能地使每个类或者成员不被外界访问.
创建对象时的流程如下:
  1. 加载指定的字节码文件进内存
  2. 通过new在堆内存中开辟空间并分配首地址值
  3. 对象中的属性进行默认初始化
  4. 调用对应的构造函数. 构造函数压栈
  5. 隐式执行super()访问父类构造函数
  6. 对象中的属性进行初始化
  7. 调用类中的构造代码块
  8. 执行构造函数中自定义的初始化代码
  9. 初始化完毕, 将地址赋给指定的引用

Java 内存分配

Java对内存空间的划分为五部分:
栈,堆,方法区,本地方法区,寄存器.

栈

栈内存中存储的是局部变量,一旦变量生命周期结束,该变量就被释放.

并不是数据结构中的栈.

堆

堆内存中存储的是实体(对象),每一个实体都有一个首地址值.new 会在堆内存中开辟空间.

堆内存中的变量都有一个默认初始化值.
