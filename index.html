Java 面向对象编程

面向对象三个特征:封装,继承和多态.

封装

封装就是隐藏实现细节. 对外提供公共的访问方式(接口).

封装的目的是为了解耦, 降低耦合可以加快系统的开发速度, 减轻维护的负担. 还提高了软件的可重用行,
也便于进一步优化.

  • 访问控制机制. 实体的可访问性由该实体声明所在的位置,以及该实体声明中所出现的访问修饰符
    (private,protected,public)共同决定. 第一规则是尽可能地使每个类或者成员不被外界访问.

继承

  1. 提高了代码的复用性
  2. 让类与类之间产生了关系,为第三个特征多态提供了前提

Java支持单继承, 不直接支持多继承. 多继承的的弊端是方法调用具有不确定性,因为方法的主体不同,Java对其进行了改良;

当事物存在(is-a)的关系时, 可以通过继承实现这个关;

子父类出现的相同名字的成员变量, 用super关键字来区分;

子类每一个构造函数第一行都有隐式的super();
如果父类没有空参数构造函数, 则需要显式调用super的方式调用父类构造函数;
并且只能定义在第一行, 因为父类的初始化要先完成;
super()和this()不能同时出现,因为他们只能定义在第一行;

多态

创建对象时的流程如下:
  1. 加载指定的字节码文件进内存
  2. 通过new在堆内存中开辟空间并分配首地址值
  3. 对象中的属性进行默认初始化
  4. 调用对应的构造函数. 构造函数压栈
  5. 隐式执行super()访问父类构造函数
  6. 对象中的属性进行初始化
  7. 调用类中的构造代码块
  8. 执行构造函数中自定义的初始化代码
  9. 初始化完毕, 将地址赋给指定的引用

Java 内存分配

Java对内存空间的划分为五部分:
栈,堆,方法区,本地方法区,寄存器.

栈

栈内存中存储的是局部变量,一旦变量生命周期结束,该变量就被释放.

并不是数据结构中的栈.

堆

堆内存中存储的是实体(对象),每一个实体都有一个首地址值.new 会在堆内存中开辟空间.

堆内存中的变量都有一个默认初始化值.

数据类型 初始值
int 0
double 0.0
boolen false
char ‘\u0000’

Java 数组

一维数组 定义方式:

元素类型[] 数组名 = new 元素类型[元素类型或数组长度];

声明变量是在栈内存中开辟空间 –> new 在堆内存中开辟空间 –> 赋值是将堆内存中
实体的首地址赋给栈内存中的变量.所以数组是引用类型.

数组编译时不会在堆分配空间,所以检查不出下标越界错误.

注意, 长度非零的数组总是可变的,所以类具有公共的静态final数组域, 或者返回这种域的访问方法,这几乎总是可变的.

二维数组 定义方式:

元素类型[][] 数组名 = new 元素类型[一维长度][二维长度];

在堆内存中一维实体存二维实体的首地址.

如果二维数组长度不定,需要分别初始化二维数组.

数组的工具类Array;

二分查找Array.binarySearch(),查询成功返回正索引,查询失败返回-(1+插入位置);

Java 基本数据类型

Java 中基本数据类型分整型,浮点型,字符型,布尔型四类八种 ;

整型
数据类型 长度 表示范围
byte 8 -128 ~ 127
short 16 -2^15 ~ (2^15)-1
int 32 -2^31 ~ (2^31)-1
long 64 -2^63 ~ (2^63)-1
浮点型
数据类型 长度
float 32
double 64
字符型
数据类型 长度
char 16
布尔型
数据类型 内容 长度
boolean true or false 1字节

知识点

  1. Java低字节向高字节转换具有自动提升性质,转换顺序为 byte->short->int->long->float->double,但高字节向低字节强制转换可能丢失精度.

  2. 默认浮点类型为double,float加后缀f或F.
